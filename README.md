# XPortsProgress Privacy Policy

This privacy policy applies to the XPortsProgress app (hereby referred to as
"Application") for mobile devices that was created by Cicero Hellmann Ratti
(hereby referred to as "Service Provider") as a fitness, training, and workout
planning application. This service is intended for use "AS IS".

## Information Collection and Use

The Application is designed to help you plan, run, and review training sessions.
The Application stores training and app data locally on your device. This may
include:

- workout plans, saved trainings, exercises, sets, repetitions, weights,
  rest timers, interval timers, recovery state, and training history;
- fitness-related information that you choose to enter, such as body weight,
  exercise preferences, training notes, set difficulty, and recovery status;
- profile information that you choose to enter, such as name, email address,
  body weight, and gender;
- notes, tasks, logs, device records, app settings, theme preferences, backup
  settings, and optional AI settings.

The Application does not use advertising SDKs, analytics SDKs, or crash
reporting SDKs. The Application does not sell personal information.

The Application is not a medical device and is not intended to diagnose, treat,
or prevent any medical condition.

## Local Storage

Application data is stored in the Application's private storage on your device.
Other applications should not be able to access this storage through normal
operating system controls.

You can delete locally stored data by deleting data inside the Application where
that option is available, or by uninstalling the Application from your device.

## Fitness and Health-Related Data

The Application may store information related to your workouts, training
progress, recovery, body weight, and exercise performance. This information is
used to provide the Application's training planning, workout session, timer,
recommendation, and progress features.

The Application does not share this data with the Service Provider unless you
choose to use an optional feature that sends data to a server or third-party
service configured by you.

## Data Deletion and User Control

You can delete locally stored Application data by using deletion features inside
the Application where available, clearing the Application's storage through your
device settings, or uninstalling the Application.

If you configure an optional backup server or third-party service, deleting data
from the Application may not automatically delete copies held by that server or
service. You should use that server's deletion tools or contact the operator of
that server for deletion requests.

The Application does not currently provide a Service Provider-operated account
system. If a future version adds account creation, this policy will be updated
to describe account deletion options.

## Optional Backup and Network Features

The Application may include optional backup or synchronization features. If you
choose to configure a backup server or network endpoint, the Application may
send workout, profile, notes, tasks, logs, device records, settings, and other
backup data needed for that backup or synchronization to the server URL that you
provide.

The Service Provider does not operate a default cloud backup service for the
Application. Any server address, token, or backup destination that you configure
is controlled by you or by the party operating that server.

## Optional AI Features

The Application may include optional AI-related settings or features. If you
enable an AI feature, provide an API key, or submit content to an AI feature,
the Application may send the information needed to complete that request to the
AI service configured for that feature.

AI features are optional. Do not enter sensitive information into an AI feature
unless you are comfortable sending that information to the configured AI service
for processing. The Service Provider does not use AI features to train a model
owned by the Service Provider.

## Permissions

The Android phone Application currently does not request special Android runtime
permissions in its manifest. If future versions add optional capabilities such
as Bluetooth, sensors, files, camera, notifications, or network-backed features
that require permissions, those permissions will be used only for the feature
that requires them.

## Third Party Access

The Application does not transmit data to advertising networks, analytics
providers, or crash reporting services.

The Service Provider may disclose information only:

- as required by law, such as to comply with a subpoena or similar legal process;
- when they believe in good faith that disclosure is necessary to protect their
  rights, protect your safety or the safety of others, investigate fraud, or
  respond to a government request.

## Data Retention Policy

Data stored locally remains on your device until you delete it, clear the
Application's storage, or uninstall the Application. Data sent to any optional
backup server or third-party service is controlled by the retention rules of
that server or service.

## Children

The Service Provider does not use the Application to knowingly solicit data from
or market to children under the age of 13. The Application is not directed to
children under 13.

## Security

The Service Provider is concerned about safeguarding your information. Local
data is protected by your device's operating system and security settings. If you
configure optional backup or synchronization, you are responsible for choosing a
trusted server and protecting any access token or credentials used with it.
